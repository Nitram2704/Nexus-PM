<task type="auto">
  <name>Task 2: Frontend API client with JWT interceptors</name>
  <files>frontend/src/api/client.ts, frontend/src/api/auth.ts, frontend/src/types/auth.ts</files>
  <action>
1. `types/auth.ts`: TypeScript interfaces
   ```typescript
   export interface User {
     id: string;
     email: string;
     first_name: string;
     last_name: string;
     avatar: string | null;
     bio: string;
   }
   export interface TokenPair { access: string; refresh: string; }
   export interface LoginRequest { email: string; password: string; }
   export interface RegisterRequest { email: string; password: string; password_confirm: string; first_name: string; }
   ```

2. `api/client.ts`: Axios instance
   - Base URL from `VITE_API_URL`
   - Request interceptor: attach `Authorization: Bearer {access_token}` from localStorage
   - Response interceptor: on 401, try refresh token. If refresh fails → logout and redirect to /login.
   - Queue pending requests during refresh (avoid race conditions)

3. `api/auth.ts`: Auth API functions
   ```typescript
   export const authApi = {
     register: (data: RegisterRequest) => client.post<TokenPair>('/auth/register/', data),
     login: (data: LoginRequest) => client.post<TokenPair>('/auth/login/', data),
     refreshToken: (refresh: string) => client.post<{ access: string }>('/auth/token/refresh/', { refresh }),
     getProfile: () => client.get<User>('/auth/profile/'),
     updateProfile: (data: Partial<User>) => client.put<User>('/auth/profile/', data),
     requestPasswordReset: (email: string) => client.post('/auth/password-reset/', { email }),
   };
   ```
  </action>
  <verify>
- Axios instance adds auth header to requests
- 401 responses trigger token refresh automatically
- Failed refresh clears tokens and redirects to login
  </verify>
</task>
