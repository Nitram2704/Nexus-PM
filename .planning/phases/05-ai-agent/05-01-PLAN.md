  <action>
`ClaudeClient` class:
- Wraps `anthropic.Anthropic` client
- Methods:
  - `generate(system_prompt, user_message, model='claude-3-haiku-20240307')` → returns text
  - `generate_json(system_prompt, user_message, model)` → returns parsed JSON (with retry on parse failure)
- Error handling: API errors, rate limits (retry with exponential backoff), timeout (30s)
- Model selection: Haiku for chat (fast), Sonnet for generation (quality)
- Fallback: if API key not set, return mock responses (for development without API key)

<task type="auto">
  <name>Task 3: Prompt templates</name>
  <files>backend/intelligence/prompts.py</files>
  <action>
Prompt templates as Python string templates with `{variable}` placeholders:

1. `GENERATE_STORIES_PROMPT`: Takes user_input, project context. Returns JSON array of stories with title, description, acceptance criteria, type, priority, story points.

2. `GENERATE_BACKLOG_PROMPT`: Takes project description. Returns JSON array of 10-20 stories organized by epic.

3. `SUGGEST_PRIORITIES_PROMPT`: Takes current backlog items. Returns reordered list with justification per change.

4. `CHAT_SYSTEM_PROMPT`: Scrum Master persona with project data injected. Responds to questions about sprint, tasks, velocity, recommendations.

5. `SPRINT_SUMMARY_PROMPT`: Takes sprint data (tasks completed/incomplete, velocity, dates). Returns structured summary with highlights, areas of improvement, recommendations.

All prompts:
- Request JSON output where applicable
- Include project name, key, and context
- Have clear role definition for the AI
- Limit output length appropriately
  </action>
</task>
