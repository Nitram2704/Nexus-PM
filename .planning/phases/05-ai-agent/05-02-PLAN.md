<objective>
Build the AI generation API: story generation, backlog generation, priority suggestion endpoints with async task pattern (submit + poll for results).

Purpose: These endpoints are the core AI value proposition — converting natural language to structured backlog items.
Output: API endpoints that trigger AI generation and return results via polling.
</objective>

<tasks>

<task type="auto">
  <name>Task 1: AI serializers</name>
  <action>
- `GenerateStoriesSerializer`: user_input (text, min 20 chars)
- `GenerateBacklogSerializer`: description (text, min 50 chars)
- `TaskResultSerializer`: task_id (Celery task ID for polling)
- `GeneratedStorySerializer`: title, description, acceptance_criteria (list), type, priority, estimated_points
- `ApproveStoriesSerializer`: stories (list of GeneratedStorySerializer objects to create as tasks)
- `AITaskStatusSerializer`: status ('pending', 'processing', 'completed', 'failed'), result (JSON)
  </action>
</task>

<task type="auto">
  <name>Task 2: AI generation views</name>
  <action>
1. `GenerateStoriesView` (APIView):
   - POST: accepts user_input, dispatches generate_stories_task.delay(), returns {task_id}
   
2. `GenerateBacklogView` (APIView):
   - POST: accepts description, dispatches generate_backlog_task.delay(), returns {task_id}
   
3. `SuggestPrioritiesView` (APIView):
   - POST: dispatches suggest_priorities_task.delay(), returns {task_id}

4. `AITaskStatusView` (APIView):
   - GET with task_id param: checks Celery AsyncResult status
   - Returns: {status, result} where result contains generated stories/priorities

5. `ApproveStoriesView` (APIView):
   - POST: accepts list of story objects, creates Task records in backlog
   - Sets is_ai_generated=True on created tasks

6. `SprintSummaryView`:
   - POST with sprint_id: generates sprint summary via AI

URLs:
```python
projects/{pid}/ai/generate-stories/
projects/{pid}/ai/generate-backlog/
projects/{pid}/ai/suggest-priorities/
projects/{pid}/ai/task-status/{task_id}/
projects/{pid}/ai/approve-stories/
projects/{pid}/ai/sprint-summary/{sprint_id}/
```
  </action>
</task>

<task type="auto">
  <name>Task 3: Frontend AI API client</name>

```typescript
export const aiApi = {
  generateStories: (projectId: string, userInput: string) =>
    client.post<{ task_id: string }>(`/projects/${projectId}/ai/generate-stories/`, { user_input: userInput }),
  generateBacklog: (projectId: string, description: string) =>
    client.post<{ task_id: string }>(`/projects/${projectId}/ai/generate-backlog/`, { description }),
  suggestPriorities: (projectId: string) =>
    client.post<{ task_id: string }>(`/projects/${projectId}/ai/suggest-priorities/`),
  getTaskStatus: (projectId: string, taskId: string) =>
    client.get<AITaskStatus>(`/projects/${projectId}/ai/task-status/${taskId}/`),
  approveStories: (projectId: string, stories: GeneratedStory[]) =>
    client.post(`/projects/${projectId}/ai/approve-stories/`, { stories }),
  generateSprintSummary: (projectId: string, sprintId: string) =>
    client.post<{ task_id: string }>(`/projects/${projectId}/ai/sprint-summary/${sprintId}/`),
};
```
  </action>
