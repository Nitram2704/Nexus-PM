<objective>
Build the Notifications API: auto-triggers, CRUD, preferences, and signals for event-driven notification creation.

Purpose: Notifications keep the team informed about important events without manual checking.
Output: Notifications created automatically on key events, API returns and manages them.
</objective>

<tasks>

<task type="auto">
  <name>Task 1: Notification serializers and views</name>
  <action>
1. Serializers:
   - `NotificationSerializer`: id, type, title, message, is_read, link, project (nested mini), created_at
   - `NotificationPreferenceSerializer`: task_assigned, sprint_deadline, ai_recommendation, mention (all booleans)

2. Views:
   - `NotificationListView` (ListAPIView): user's notifications, filterable by is_read. Returns unread_count in response header.
   - `MarkReadView`: POST with notification_ids OR "all" to mark as read
   - `NotificationPreferenceView` (RetrieveUpdateAPIView): GET/PUT preferences
   - `UnreadCountView`: GET returns { count: N }

3. URLs: `/api/v1/notifications/`
  </action>
</task>

<task type="auto">
  <name>Task 2: Auto-trigger signals</name>
  <files>backend/notifications/signals.py</files>
  <action>
Signals that create notifications:

1. `task_assigned_signal` (post_save on Task):
   - When assignee changes and new assignee is not null
   - Creates notification for assignee: "Te asignaron {task_key}: {task_title}"
   - Link: `/project/{project_id}/backlog/{task_id}`
   - Respects NotificationPreference.task_assigned

2. `sprint_deadline_signal` (Celery periodic task, runs daily):
   - Checks all active sprints where end_date - today <= 2 days
   - Creates notification for all members: "Sprint {name} vence en {N} dĂ­as. Quedan {count} tareas."
   - Only creates once per sprint (check existing notification)
   - Respects NotificationPreference.sprint_deadline

3. `ai_recommendation_signal`:
   - Called from AI tasks when generating recommendations
   - Creates notification: "Nueva recomendaciĂłn del Agente IA para {project}"
   - Respects NotificationPreference.ai_recommendation
  </action>
</task>

<task type="auto">
  <name>Task 3: Frontend API</name>
  <action>
```typescript
export const notificationsApi = {
  list: (isRead?: boolean) => client.get<Notification[]>('/notifications/', { params: { is_read: isRead } }),
  markRead: (ids: string[]) => client.post('/notifications/mark-read/', { notification_ids: ids }),
  markAllRead: () => client.post('/notifications/mark-read/', { all: true }),
  getUnreadCount: () => client.get<{ count: number }>('/notifications/unread-count/'),
  getPreferences: () => client.get<NotificationPreference>('/notifications/preferences/'),
  updatePreferences: (data: Partial<NotificationPreference>) =>
    client.put<NotificationPreference>('/notifications/preferences/', data),
};
```
  </action>
